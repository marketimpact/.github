# Contributing

This is a MarketImpact repository. Keep changes small, scoped, and consistent with the local project instructions.

## Workflow

1. Create a branch from the default branch.
2. Use TDD for new features and bug fixes where code is involved.
3. Run the focused checks for the touched project before opening a PR.
4. Open a PR to the default branch; active repos are protected by a cost-safe ruleset.

## Guardrails

- Do not commit secrets or `.env` files.
- Do not add dependencies unless the relevant plan explicitly allows them.
- Update project memory/docs when an operational invariant changes.
