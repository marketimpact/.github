# Security Policy

## Reporting a Vulnerability

Report suspected vulnerabilities privately to `thomas@marketimpact.org`.
Do not open a public issue with secrets, credentials, exploitable details, or production data.

Include the affected repository/app, reproduction steps or logs, relevant commit SHA or URL, and any known exposure window.

Security fixes should go through a private branch or GitHub security advisory when disclosure risk requires it. Rotate exposed credentials before merging a public remediation.
